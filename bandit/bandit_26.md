## Level goal
Logging in to bandit26 from bandit25 should be fairly easyâ€¦ The shell for user bandit26 is not **/bin/bash**, but something else. Find out what it is, how it works and how to break out of it.

## Solution
    - to check the shell used by the user bandit26, i need to go to:
    	- **cat /etc/passwd | grep bandit26** => /usr/bin/showtext
    - **cat /usr/bin/showtext**

    ```
    bash
    #!/bin/sh

    export TERM=linux

    more ~/text.txt
    exit 0
    ```

    - this opens a text file with the "more" program
    - if the message printed from the text file with more does not fit in the terminal window, we can use the command mode of the "more" command to run our own commands
    - to connect to the bandit26 custom shell, we can use the private key from the bandit25 user, which we copy on our own machine and change the privilege to **chmod 700 bandit26.sshkey**
    - then we make the window as small as possible and we use **ssh -i bandit26.sshkey bandit26@bandit.labs.overthewire.org -p 2220**
    - then we type ":v" to enter command more
    - then **"e /etc/bandit_pass/bandit26**
